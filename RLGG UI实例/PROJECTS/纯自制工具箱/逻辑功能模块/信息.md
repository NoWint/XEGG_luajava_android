# RLGG常见错误完整指南

## 1. 空指针异常 (NullPointerException)

### 🚨 **错误描述**

```
vm error: java.lang.NullPointerException: 
Attempt to invoke virtual method 'boolean ۟.̅.j()' on a null object reference
```

### 📋 **常见原因**

- UI元素未正确初始化就被访问
- Activity/Context对象为null
- LuaJava对象生命周期问题
- 获取View元素失败但未检查null值

### 🔧 **解决办法**

```lua
-- ❌ 错误写法
local button = window.someButton
button:setText("文本")  -- 如果button是nil会报错

-- ✅ 正确写法
local button = window.someButton
if button then
    button:setText("文本")
else
    print("按钮获取失败")
end

-- ✅ 使用pcall包装
local success, err = pcall(function()
    button:setText("文本")
end)
if not success then
    print("设置失败:", err)
end
```

## 2. 布局加载错误 (Layout Error)

### 🚨 **错误描述**

```
loadlayout error: First value Must be a Class, checked import package
```

### 📋 **常见原因**

- 使用了RLGG不支持的Android类
- MaterialCardView、android.fix.TextView等自定义类
- 类名拼写错误或版本不兼容

### 🔧 **解决办法**

```lua
-- ❌ 错误写法
{
    com.google.android.material.card.MaterialCardView,  -- 不支持
    android.fix.TextView,  -- 不支持
}

-- ✅ 正确写法  
{
    LinearLayout,  -- 使用标准组件
    TextView,      -- 使用标准组件
    background = "#FFFFFF",
    elevation = "2dp"  -- 用属性模拟Material效果
}

-- ✅ 检查类是否可用
local function safeBindClass(className)
    local success, clazz = pcall(luajava.bindClass, className)
    return success and clazz or nil
end
```

## 3. 类绑定失败 (Class Binding Error)

### 🚨 **错误描述**

```
attempt to call a nil value (global 'getCurrentActivity')
java.lang.ClassNotFoundException
```

### 📋 **常见原因**

- 使用了不存在的API函数
- Android类路径错误
- RLGG版本不支持某些API

### 🔧 **解决办法**

```lua
-- ❌ 错误写法
local activity = getCurrentActivity()  -- 该函数不存在

-- ✅ 正确写法
local context = nil
if rl and rl.getContext then
    context = rl.getContext()
elseif rl and rl.getActivity then
    context = rl.getActivity()
end

-- ✅ 安全类绑定
local ChangeBounds = nil
local success, err = pcall(function()
    ChangeBounds = luajava.bindClass("android.transition.ChangeBounds")
end)
if not success then
    print("ChangeBounds类不可用:", err)
end
```

## 4. 权限错误 (Permission Error)

### 🚨 **错误描述**

```
悬浮窗创建失败
Permission denied
Unable to add window
```

### 📋 **常见原因**

- RLGG没有悬浮窗权限
- 系统不允许悬浮窗显示
- 在不支持的Android版本上运行

### 🔧 **解决办法**

```lua
-- 检查权限和环境
if not rl then
    print("RLGG环境不可用")
    return
end

-- 检查悬浮窗功能
if rl.newFloating then
    local success, window = pcall(rl.newFloating, layout)
    if success and window then
        print("悬浮窗创建成功")
    else
        print("悬浮窗创建失败，请检查权限")
        -- 降级到其他UI方案
        if rl.alert then
            rl.alert("提示", "需要悬浮窗权限")
        end
    end
end
```

**手动解决步骤：**

1. 进入系统设置 → 应用管理 → RLGG
1. 权限管理 → 允许”显示悬浮窗”
1. 重新运行脚本

## 5. 内存错误 (Memory Error)

### 🚨 **错误描述**

```
java.lang.OutOfMemoryError
Script execution timeout
Stack overflow
```

### 📋 **常见原因**

- 无限循环
- 大量对象创建未释放
- 递归调用过深
- 图片资源过大

### 🔧 **解决办法**

```lua
-- ❌ 可能导致内存问题
while true do
    local obj = SomeClass()  -- 不断创建对象
    -- 没有释放和break条件
end

-- ✅ 正确写法
local count = 0
while count < 100 do  -- 设置上限
    local obj = SomeClass()
    -- 处理逻辑
    obj = nil  -- 帮助垃圾回收
    count = count + 1
    
    if someCondition then
        break  -- 提供退出条件
    end
end

-- ✅ 使用延迟避免阻塞
local function processWithDelay(items, callback)
    for i, item in ipairs(items) do
        callback(item)
        if i % 10 == 0 then  -- 每10个项目延迟一下
            rl.delay(10, function() end)
        end
    end
end
```

## 7. 网络连接错误 (Network Error)

### 🚨 **错误描述**

```
网络验证失败
连接超时
云更新失败
```

### 📋 **常见原因**

- 网络连接不稳定
- 服务器维护
- 防火墙阻拦
- DNS解析失败

### 🔧 **解决办法**

```lua
-- 网络请求重试机制
local function requestWithRetry(url, maxRetries)
    maxRetries = maxRetries or 3
    
    for i = 1, maxRetries do
        local success, result = pcall(function()
            return http.get(url)  -- 假设的网络请求函数
        end)
        
        if success then
            return result
        else
            print("网络请求失败，重试", i, "/", maxRetries)
            if i < maxRetries then
                rl.delay(1000, function() end)  -- 等1秒重试
            end
        end
    end
    
    return nil
end
```

**手动解决步骤：**

1. 检查网络连接
1. 尝试切换网络(WiFi/移动数据)
1. 检查是否被防火墙拦截
1. 联系开发者确认服务器状态

## 8. 文件访问错误 (File Access Error)

### 🚨 **错误描述**

```
Permission denied (open failed: EACCES)
No such file or directory
java.io.FileNotFoundException
```

### 📋 **常见原因**

- 文件不存在
- 没有读写权限
- 路径错误
- 存储空间不足

### 🔧 **解决办法**

```lua
-- ✅ 安全文件操作
local function safeReadFile(filepath)
    if not filepath then
        print("文件路径为空")
        return nil
    end
    
    local success, content = pcall(function()
        local file = io.open(filepath, "r")
        if not file then
            error("文件打开失败: " .. filepath)
        end
        local data = file:read("*all")
        file:close()
        return data
    end)
    
    if success then
        return content
    else
        print("文件读取失败:", content)
        return nil
    end
end

-- ✅ 检查文件是否存在
local function fileExists(filepath)
    local file = io.open(filepath, "r")
    if file then
        file:close()
        return true
    end
    return false
end
```

## 9. 版本兼容性错误 (Compatibility Error)

### 🚨 **错误描述**

```
This method is not available in current Android version
API level xxx is required
Unsupported operation
```

### 📋 **常见原因**

- Android版本过低
- RLGG版本不兼容
- 使用了新API但系统不支持

### 🔧 **解决办法**

```lua
-- ✅ 版本检查
local function checkAndroidVersion()
    local Build = luajava.bindClass("android.os.Build")
    local sdkVersion = Build.VERSION.SDK_INT
    
    print("当前Android SDK版本:", sdkVersion)
    
    if sdkVersion < 21 then
        print("⚠️ Android版本过低，某些功能可能不可用")
        return false
    end
    
    return true
end

-- ✅ 功能降级处理
local function createUIWithFallback()
    if checkAndroidVersion() then
        -- 使用新API
        return createModernUI()
    else
        -- 使用兼容API
        return createLegacyUI()
    end
end
```
